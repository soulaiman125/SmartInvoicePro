# PDF System — SmartInvoice Pro

Enterprise-grade PDF generation for invoices and quotations. Phase 1 of the
premium experience upgrade.

## Overview

PDFs are produced server-side with [PDFKit](https://pdfkit.org/) by a small
layout framework in [`backend/src/services/pdf.service.js`](backend/src/services/pdf.service.js).
The public API is unchanged and backward compatible:

```js
import { buildInvoicePdf, buildQuotePdf } from './services/pdf.service.js';

const doc = buildInvoicePdf(invoice, org, { template: 'modern' }); // PDFDocument
doc.pipe(res);
doc.end();
```

`options.template` is optional and defaults to `modern`.

## Templates

Two professionally designed templates ship out of the box:

| Template  | Look & feel                              | Typography            |
| --------- | ---------------------------------------- | --------------------- |
| `modern`  | Branded gradient colour header, zebra-striped line items (default) | Helvetica family |
| `classic` | Traditional ruled layout, monochrome     | Times serif family    |

The brand colour for the `modern` band can be overridden per organization via
`org.settings.brandColor` (any hex string). Footer text can be customized via
`org.settings.invoiceFooter` / `org.settings.quoteFooter`.

## Document anatomy

Both documents share a consistent, premium layout:

- **Header** — company logo (or branded monogram fallback), organization name,
  tax ID and address; document title, number and a colour-coded **status badge**.
- **Billed-to block** — client name, email, tax ID and billing address.
- **Meta column** — document number, issue date, due/valid-until date, currency,
  and a highlighted amount-due / validity figure.
- **Line-item table** — description, quantity, unit price, **per-line tax %**,
  and amount. Long descriptions wrap; the table paginates automatically.
- **Summary** — subtotal, discount (only shown when non-zero), **tax breakdown**,
  total, and a **payment summary** (amount paid + balance due) for invoices.
- **Notes / terms** — free-text notes (invoice) or a validity statement plus an
  **acceptance / signature section** (quote).
- **Footer** — appears on every page with custom footer text, **page X of N**,
  and a "Generated by SmartInvoice Pro" line.
- **Status stamp** — a subtle diagonal `PAID` / `OVERDUE` / `CANCELLED` watermark
  reflects the document state at a glance.

## Logo handling

`tryDrawLogo()` embeds the org logo when `org.logoUrl` is a **data URI** or an
**absolute file path**. Remote `http(s)` URLs are intentionally not fetched
synchronously during rendering; in that case the branded monogram is drawn
instead. To use a hosted logo, store it as a base64 data URI in settings.

## Print quality & layout

- A4 page size, 50 pt margins, 495 pt content width.
- Vector text and shapes throughout — crisp at any zoom and print resolution.
- Currency is formatted with `Intl.NumberFormat` (locale-aware symbols).
- **Automatic pagination**: the item table and summary block detect overflow and
  start a new page, re-drawing the table header; footers are rendered across all
  pages via PDFKit buffered pages.

## API endpoints

| Method | Path                              | Notes                          |
| ------ | --------------------------------- | ------------------------------ |
| GET    | `/api/v1/invoices/:id/pdf`        | `?template=modern\|classic`    |
| GET    | `/api/v1/quotations/:id/pdf`      | `?template=modern\|classic`    |

Both require authentication and stream `application/pdf` inline.

## Frontend

The [`PdfMenu`](frontend/src/components/PdfMenu.jsx) component renders a
**Download PDF** button with a template picker (Modern / Classic) and a busy
state. It is wired into the invoice and quote detail pages. Failed downloads
surface a toast.

## Verification

The engine was validated against seeded data for:

- Invoice — `modern` and `classic` templates
- Overdue invoice — status stamp rendering
- 100+ line items — multi-page pagination with repeated headers and footers
- Quote — `modern` and `classic` with acceptance section

All outputs are valid `%PDF-` documents. Frontend `lint` and `build` pass.

## Extending

To add a template, add an entry to `TEMPLATES` in `pdf.service.js` (accent,
fonts, header style, zebra) and to the `TEMPLATES` list in `PdfMenu.jsx`. The
layout engine is template-driven, so no further changes are required.
